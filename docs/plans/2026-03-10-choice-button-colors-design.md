# 選択肢背景色 Design

## Goal

Issue #3 `選択肢に背景色を適用する` を、3歳児が見分けやすく、親が色で案内しやすい UI として追加する。

## Decision

- 4択ボタンは `左上=みずいろ / 右上=ももいろ / 左下=みどり / 右下=きいろ` の位置固定色にする
- ボタンの背景色は通常時に保持し、`correct / wrong / disabled` は枠線・彩度・不透明度で重ねて見分ける
- 正解時は背景色を消さずにアクセント枠とシャドウを強める
- 不正解時はグレーアウトし、再選択不可にする
- スマホ縦は既存の 2x2 を維持し、極小幅だけ 1 列フォールバックのままにする

## Why

- 親が「ひだりうえ の みずいろ」など位置と色の両方で案内できる
- 正解色を全面上書きすると、元の色による案内文脈が消える
- 現状実装は `js/app.js` でボタン生成時に属性を付けるだけで済み、変更が小さい
- `styles.css` 側でベース色と状態色を分離すると、今後の見た目調整もしやすい

## Visual Rules

### Base Colors

- `slot-0`: `#e1f4fe`
- `slot-1`: `#fce7f3`
- `slot-2`: `#dcfce7`
- `slot-3`: `#fef9c3`
- 文字色は既存の `--ink` を維持する

### State Layers

- 通常: 位置固定色 + 白寄りの外枠 + 共通シャドウ
- hover / focus: 少し浮き上がる + 枠線とシャドウを少し強める
- correct: 背景色は維持しつつ、オレンジ系の強い枠線とシャドウを適用する
- wrong: グレー寄りに落として不透明度を下げる
- disabled: 正解後に正解以外の彩度を落とし、押せないことを明示する

## Scope

- `js/app.js` で各選択肢に位置スロット属性を付ける
- `styles.css` で 4 色のベーススタイルと状態レイヤを追加する
- CSS ベースの見分けやすさを固定する回帰テストを追加する
- README に選択肢の色分けを追記する

## Non-Goals

- 問題数や進行ルールの変更
- 正解演出のアニメーション追加
- 音声ガイドや読み上げ機能の追加
