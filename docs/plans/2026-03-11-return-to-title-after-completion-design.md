# 完了後タイトル復帰 Design

## Goal

5問完了後の再挑戦導線をやめ、達成感を残したままタイトル画面へ戻れるようにする。

## Decision

- 完了画面自体は維持する
- 完了画面の CTA は `🏠 タイトルへ もどる` に変更する
- CTA 押下時は新しいラウンドを始めず、タイトル画面へ戻す
- 完了メッセージは `たいへん よくできました 💮` に変更して再挑戦誘導を消す
- HTML/CSS は原則変更せず、`js/app.js` とテストの最小差分で実装する

## Why

- 完了直後にタイトルへ即時遷移させるより、子どもに「おわった」が伝わりやすい
- 既存の完了画面レイアウトと CTA スタイルを再利用でき、レスポンシブ崩れのリスクが低い
- 次の開始操作をタイトル画面へ戻すことで、導線が一貫する

## Scope

- `js/app.js` の完了時文言と `handleNext()` の完了時分岐を変更する
- DOM テストで「完了後ボタン -> タイトル復帰」を固定する
- 必要に応じて README の導線説明を微修正する

## Non-Goals

- 完了画面の大幅なレイアウト変更
- タイトル画面デザインの再調整
- 自動遷移やタイマー導入
