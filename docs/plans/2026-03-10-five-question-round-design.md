# 5問固定ラウンド Design

## Goal

`#2 1回の出題を5問にする` を、既存 UI の見た目を崩さず最小差分で実現する。

## Decision

- 1プレイは `data/trains.json` 全体からランダムに 5 問だけ選ぶ
- 進捗表示は `1 / 5` 形式にする
- 既存の完了表示 `ぜんぶ できたね！` と `もう いっかい` はそのまま使う
- HTML/CSS は触らず、出題順生成ロジックを `js/app.js` 中心に変更する

## Why

- 既存実装は `state.order.length` を進捗表示と終了判定の基準にしている
- そのため、出題順配列を 5 件に制限すれば、UI や完了フローを大きく変えずに要件を満たせる
- 将来 `結果画面` issue を実装するときも、`1プレイ = state.order` の前提を維持できる

## Scope

- `js/round.js` を新設し、ラウンド用の ID 抽選ロジックを切り出す
- `js/app.js` で初回起動時と再プレイ時に 5 問ラウンドを生成する
- README に 1 プレイ 5 問を追記する
- Node テストで 5 問抽選ロジックを固定する

## Non-Goals

- 結果画面の UI 強化
- タイトル画面の追加
- 不正解後の進行ルール変更
