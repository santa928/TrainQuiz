# 結果画面 Design

## Goal

5問終了後に、3歳児でも達成感が分かる結果画面を表示し、`せいかい数` と次の導線をやさしく見せる。

## Decision

- 既存の `quiz-screen` を結果画面として拡張し、別画面は増やさない
- 結果状態では画像・選択肢・出典・`つぎへ` を隠し、中央に結果パネルを表示する
- 結果パネルには `5もんちゅう` と `〇もん せいかい！` を大きく出す
- CTA は `もういちど あそぶ` を主導線、`タイトルへ もどる` を副導線として並べる
- スマホは縦積み、タブレットも中央寄せの縦積みを維持して誤タップを避ける

## Why

- 既存の2画面構成を保てるため、状態管理と DOM 差分が小さい
- すでにある完了画面の文脈を活かしつつ、Issue #7 の `せいかい数` と 2 導線を足せる
- 3歳児向けでは情報量を増やしすぎず、中央の大きな結果と大きなボタンを優先したほうが伝わりやすい

## Scope

- `index.html` に結果パネルと 2 本のボタンを追加する
- `styles.css` に結果状態の表示切り替えと結果パネルのスタイルを追加する
- `js/app.js` に `せいかい数` 集計、結果状態表示、`もういちど` / `タイトルへ` ハンドラを追加する
- `tests/app.test.mjs` で結果画面表示と 2 本の導線を固定する
- README の機能説明を結果画面に合わせて更新する

## Non-Goals

- タイトル画面の再設計
- アニメーション追加や自動遷移
- 複雑な称号・ランク表示
