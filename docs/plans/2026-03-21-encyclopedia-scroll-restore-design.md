# 図鑑一覧スクロール復元 Design

## Goal

Issue `#11 図鑑で一覧に戻るとスクロールが毎回1番上に戻ってしまう問題` を、既存の図鑑 UI を崩さず最小差分で解消する。

## Decision

- 詳細画面へ入る直前に、図鑑一覧の `scrollTop` とページ全体の `window.scrollY` を state へ保存する
- 詳細から一覧へ戻るときだけ保存値を使って一覧スクロール位置を復元する
- タイトルから図鑑を開くときは従来どおり一覧先頭から始める
- 一覧の見た目や文言は変更せず、状態管理と戻り導線だけを調整する

## Why

- 現状の issue は「詳細から戻ると先頭に戻る」挙動だけが問題で、永続保存や UI 再設計までは不要
- 一覧は `showEncyclopediaList()` 呼び出しごとに再描画されるため、明示的に復元値を持つほうが挙動を固定しやすい
- タイトルに戻ったあとまで位置を残すと「いま開いた一覧」の期待とずれる可能性があるため、復元範囲は詳細→一覧に限定する

## Scope

- `js/app.js` に図鑑一覧のスクロール復元 state と戻り処理を追加する
- `tests/app.test.mjs` に scroll 復元の失敗テストを追加する
- `docs/plans/` に本設計と実装計画を残す

## Non-Goals

- 図鑑レイアウトやカードデザインの変更
- タイトルへ戻ったあとも位置を永続化すること
- URL, history, storage を使った画面状態の復元

## Behavior

- タイトル -> 図鑑一覧: 一覧は先頭から表示する
- 図鑑一覧 -> 詳細: 現在の一覧 `scrollTop` と `window.scrollY` を保存する
- 詳細 -> 図鑑一覧: 一覧再描画後に保存済みの scroll を復元する

## Risks and Mitigations

- テスト用 Fake DOM に page scroll API が不足している: `document.defaultView.scrollY` / `scrollTo()` を最小追加して挙動を観測できるようにする
- 戻り直後の再描画でブラウザ側スクロールが遅れて走る: 即時復元に加えて、数フレーム後と短い timeout でも再適用して安定化する
